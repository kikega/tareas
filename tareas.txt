Aplicación para la gestion de proyectos, tareas de cada uno y subtareas:
La aplicación, será para un solo usuario cuyo id será el correo electrónico, por lo que se deberá extender el modelo de usuario para añadir el correo electrónico.
Se utilizará django para el backend, si fuera necesario se podrá utilizar astro para el frontend, con tallind para el estilos y pyhton para la base de datos, que será postgresql.
Se utilizará un estilo con colores suaves y agradables, con un diseño minimalista y moderno y en modo dark.
La aplicación constará de las siguientes caracteristicas:

- Login
- Dashboard
- Proyectos
- Tareas
- Subtareas
- Categorias
- Etiquetas
- Calendario

La estructura de proyectos y tareas será la siguiente:
- Un proyecto puede tener muchas tareas.
- Una tarea puede tener muchas subtareas.
- Cada subtarea pertenece a una tarea y cada tarea pertenece a un proyecto.
- Cada subtarea debe estar relacionada con el correo electrónico del usuario.
- Cada tarea debe estar relacionada con el correo electrónico del usuario.
- Cada proyecto debe estar relacionado con el correo electrónico del usuario.
